Scheme

NexT主题下有4种子主题可供选择，需要在/themes/next/config.yml主题配置文件中修改。在Scheme Settings处，将默认的Muse前面用’#’注释掉，把Pisces前的#去掉，即本博客使用的主题。当然你可以尝试其他的风格并选择自己心仪的

菜单栏

同样是修改/themes/next/config.yml配置文件，在menu节点，默认启用的菜单项是home、tags、categories、archives，当然此时在你的博客上点击标签（tags）和分类（categories）还不能正常跳转，因为没有相应的资源文件，需要自己建立对应的文件夹，详见2.1

功能

Tag & Category

GitShell进入你的hexo安装目录，输入命令 hexo new page tags,此处的tags不是普通的新建文章的标题，而是以关键字的形式存在的，因为之后菜单的“标签”链接就是对应的此处的资源文件，执行命令后发现Hexo/source文件夹中多出了一个tags文件夹。

INFO Created: E:\Hexo\source\tags\index.md

文本编辑器打开该文件夹中的index.md文件，添加type节点。

如若添加了评论系统，可以在这页加上 comment: false

添加categories同理。

在你的新文章中：tags: - … - ..
categories: - … - …

注：1. 菜单栏的其他选项也可以用同样的方法创建内容
  2. 在你删掉tags时，如若发现标签数统计不对，可以hexo clean

文章阅读量

创建应用，比如应用名是View，创建完成后点击View，在存储中点击创建Class，Class名称填Counter，完成创建Class。在设置中点击应用Key，在/themes/next/_config.yml中搜素leancloud_visitors，将App ID和App Key相应替换#<app_id>和#<app_key>

在设置中点击安全中心，在web安全域名填入博客的网址
